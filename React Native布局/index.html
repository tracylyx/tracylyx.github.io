React Native布局

最近做的一个项目，使用的是RN，我在项目中主要负责卡片的绘制。这就涉及到和样式、flex布局相关的内容，看似容易，当时对于好久好久好久没写样式的我来说，还是稍稍有些困难哒。
自己给自己挖了好多坑，例如，没有提前认认真真的思考过内部布局就开始绘制，导致后边在app中跑的时候，有些乱。

RN中的flex布局

简单概念描述

flex布局中有主轴、交叉轴、容器、项目这些名词。

主轴、交叉轴是相对的，在RN下，默认垂直方向为主轴，从而可知水平方向为交叉轴。

容器：设定flex属性的元素。

项目：被容器包裹的元素。

场景 水平 垂直
RN默认 交叉轴 主轴
flexBox布局 主轴 交叉轴
容器属性

  • flexDirection 决定主轴的方向，即项目的排列方向

    • row：默认值，主轴为水平方向，由左至右

    • row-reverse：主轴为水平方向，由右至左

    • column：主轴为垂直方向，由上至下

    • column-reverse：主轴为垂直方向，由下至上

  • flexWrap 用来定义项目在一行中显示的处理方式

    • nowrap：默认值，不换行，超过部分会被遮挡
    • wrap：换行，超过部分这行从左至右显示
    • wrap-reverse：换行，展示由下至上 从左至右展示

  • justifyContent 定义项目在主轴上的对齐方式，与轴的方向有关，假设主轴从左至右

    • flex-start：默认值，左对齐
    • flex-end：右对齐
    • center：居中
    • space-between：两端对齐，项目之间的间隔相等
    • space-around；每个项目两侧的间隔相等。所以项目之间的间隔比项目与边框的间隔大一倍

  • alignItems 定义项目在交叉轴上的对齐方式，与交叉轴的方向有关，假设交叉轴从上到下

    • flex-start：交叉轴的起点对齐
    • flex-end：交叉轴的终点对齐
    • center：交叉轴的中点对齐
    • baseline：项目的第一行文字的基线对齐
    • stretch：默认值，如果项目未设置高度或者为auto，将沾满整个容器的高度

RN中的写法

RN开发中遇到的问题集合

  • 组件名字首字母一定要大写

  • return 中只能包含一个标签，否则报错

  • justifyContent alignItems 是容器的属性，因此设置只对父元素其效果，用于界定子组件的位置。

  • flex: 1; 用来确保内容撑满当前容器

  • 实现文本溢出展示…的前提需要设定容器的宽度，否则无效

  • 无法使用borderStyle: ‘dashed’ 设置某一边的样式，但是设置solid可以
