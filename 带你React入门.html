带你React入门

学一个记一个，帮助自己也服务他人
（假如文章有幸被正在准备学习React的同学看大的话～）。

React

  • React 不是一个框架，它像是 MVC 中的V，用来高效处理 UI。
  • React 可以看作是简单的 UI 状态机，当状态发生变化时，React 以最有效的方式更新 UI(kepp the interface up-to-data when the data changes)。
  • React 有自己处理冒泡（bubble）和捕获（capture）的方式，确保事件对象（event）传递。
  • 在 React 中，标签具有 self-closing 属性，因此这样的
    等价于
  • key、ref、dangerouslySetInnerHTML 在 React 中特有
  • 不像其他构建web应用的框架，React 不使用模板，而是通过可用性强、灵活的组件创建页面。

JSX

  • js 的扩展，像 XML。
  • 可以使用在 React 中（在 React 中不是必须的，你完全可以使用原生 js 是实现树状结构）
  • 在 JSX 中使用 if…else…结构的方式

组件 Component

  • 写组件的核心思想：单一职责 single responsibility priciple
  • 组件，可用性强
  • 尽可能保持组件的无状态（无state）
  • 常用模式：
    • 创建几个无状态的组件（仅用来渲染数据）
    • 通过一个有状态的组件包裹、管理
  • 组件的优点：
    • 使用 js 语言，灵活且强大
    • 易扩展和维护
    • 避免 xss 攻击

props VS state

  • props 当值设定后，在组件中就不可改变了。
  • state 相应用户、服务请求、当发生改变时，state也跟着变化，然后通知到UI，从而做界面的更新。
