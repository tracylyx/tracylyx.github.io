npm相关笔记

碎碎念

很多东西看的时候，明白。但是，过后没多久就会遗忘。我总结了两点原因：1⃣ ️没有具体用学到的内容去实践；2⃣️ 没有做属于自己的笔记。通过自己习惯的方式记录下它，每当需要用到的时候，像手册一样，可以去翻阅，进而加深印象，抛弃笔记。

npm

node package manager，node的包管理工具。
特点：
1⃣️ 便于代码共享，知道包名，npm install 即可
2⃣️ 代码重用
3⃣️ 代码版本管理

npm 安装

通过 npm 安装完项目项目中需要的包之后，会在根目录下自动生成一个node_module的文件夹，安装的的包自然都会在里边

本地包管理方式

通常会使用 package.json 的方式管理本地安装的包，这样可以方便快捷的将项目构建展示在其他开发面前。方便查找项目安装、使用的包的情况。

有一点需要注意的是，这个文件 package.json 是json文件，因此，里边不能含有注释，否则报错。

创建 package.json 文件

国内访问 npm 包仓库拉取信息的过程，有可能很慢很慢，这种情况下，我们可以使用淘宝镜像。需要注意下，尽量避免使用 cnpm 的命令去下包包，因为通过这种方式下载的包的路径会比较奇怪（我是看其他文章中提到的）。

安装包的时候有两种方式，一种是保存在正常依赖上，还有一种是保存在开发依赖上（我的理解是项目中需要使用的一个打包工具、单元测试等包放置在此处）。命令上有一丢丢不同。

创建自己的包

你不仅是 npm 包的使用者，同时，如果你有兴趣的话，也可以称为 npm 的供应方。

你可以将自己写好的 npm 包（⚠️ 包名一定是唯一的哦）发布到 npm 仓库上，你只需要这样做：

  • 在官网上注册个用户
  • 使用这些命令：
    • npm adduser # 添加用户
    • npm login # 登录
    • npm publish # 发布包
    • npm version # 更新的版本号，接着再publish一次即可

npm 中的包有版本的概念。版本号中每个位置的数字都有其具体的含义，因此是语义化的版本。

常用的命令

  • npm uninstall 卸载包
  • npm uninstall –save # 卸载包的同时将package.json的dependencies移除
  • npm outdated -g –depth=0 # 查看全局的包版本信息
  • npm update -g # 更新所有全局包
  • npm uninstall -g # 卸载全局指定包
